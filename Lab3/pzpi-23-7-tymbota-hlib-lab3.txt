Опис прийнятих інженерних рішень:
Для web-застосунку AutoInsurance Web було обрано односторінкову архітектуру на основі Vite та модульного JavaScript. Таке рішення дозволяє швидко запускати клієнтську частину, розділяти функціональність на окремі файли та зручно підтримувати інтерфейс без зміни backend-частини системи.
Застосунок має рольову структуру та передбачає окремі сценарії роботи для звичайних користувачів і адміністратора. Користувач із роллю Driver може переглядати страхові поліси, дані автомобіля, телеметрію та створювати заявки про страховий випадок. Роль Agent орієнтована на перегляд клієнтських даних і страхових заявок. Manager має можливість обробляти заявки та змінювати їхній статус. Для Admin реалізовано окрему адміністративну панель, у якій доступне керування користувачами, системними даними, імпортом, експортом і резервним копіюванням.
Взаємодія із серверною частиною виконується через окремий API-клієнт, який надсилає HTTP-запити до REST API системи AutoInsurance у форматі JSON. При цьому backend у межах лабораторної роботи не змінювався. Для підвищення стабільності роботи front-end частини реалізовано fallback-логіку: якщо сервер тимчасово недоступний, демонстраційні дані та зміни користувача зберігаються у локальному сховищі браузера.
Стан застосунку винесено в окремий модуль, який відповідає за поточну роль користувача, мову інтерфейсу, регіональні налаштування та основні дані системи. Це спрощує обмін інформацією між сторінками та дозволяє повторно використовувати спільну логіку в клієнтському й адміністративному інтерфейсах.
Інтерфейс підтримує українську та англійську мови, а також регіональне форматування дат, часу та валюти. Дані відображаються у вигляді інформаційних карток, списків, таблиць, форм і статусних позначок. Додатково реалізовано функції імпорту, експорту та backup даних, що дозволяє використовувати web-застосунок для демонстрації адміністрування системи без внесення змін у серверну частину.
